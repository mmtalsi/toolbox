Rapport de Test d'Intrusion

1. Introduction

Ce rapport présente les résultats du test d'intrusion effectué sur l'infrastructure de l'entreprise ExempleCorp entre le 1er et le 3 mai 2025.

2. Méthodologie

Le test a été réalisé selon la méthodologie OWASP. Il inclut les phases suivantes :

  • Collecte d'informations
  • Scan de vulnérabilités
  • Exploitation
  • Post-exploitation
  • Rapport

3. Résumé Exécutif

Le test a permis d'identifier plusieurs vulnérabilités, dont une critique. Il est recommandé de corriger ces failles dans les plus brefs délais.

4. Vulnérabilités Identifiées

Nom Gravité Description Recommandation
Injection SQL Critique Injection possible via le formulaire de login Utiliser des requêtes préparées (PDO ou ORM)
XSS Réfléchi Moyenne Paramètre URL non filtré Sanitiser la sortie HTML

5. Recommandations Générales

  • Mettre à jour toutes les bibliothèques logicielles
  • Former les développeurs à la sécurité applicative
  • Configurer un WAF (Web Application Firewall)

6. Conclusion

Le test démontre que plusieurs vecteurs d'attaque sont exploitables. Une attention urgente est requise pour corriger les failles critiques.
